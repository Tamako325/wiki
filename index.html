WaterHill + CC Switch + Codex

Codex 接入中转站使用 GPT 教程

按照三步完成配置：先在 WaterHill 中转站创建 GPT 分组密钥，再用 CC Switch 添加 OpenAI 兼容供应商，最后安装 Codex 并切换到刚配置好的供应商。

配置路径

  1. WaterHill创建 GPT 分组 API 密钥
  2. CC Switch添加 OpenAI 兼容供应商
  3. Codex安装后使用该供应商

第一部分

注册 WaterHill 中转站

打开 https://www.waterhill.cyou/， 点击“立即开始”或右上角“登录”都可以进入登录页面。没有账号时点击“注册”，优惠码可以不填。

第一部分

创建 API 密钥、选择分组并复制密钥

1

进入 API 密钥页面

登录后，在左侧菜单中点击 API 密钥。

2

点击创建密钥

在 API 密钥页面点击右上角“创建密钥”，准备进入分组选择流程。

3

选择 GPTfree 和混合分组

进入选择分组页面后，主分组和副分组分别选择 GPTfree 和 混合 即可，一个选 GPTfree，一个选混合。

4

复制生成的密钥

密钥创建成功后，回到 API 密钥列表，点击复制按钮保存密钥。后面配置 CC Switch 时会用到它。

第二部分

下载 CC Switch

优先打开 CC Switch 官网下载。如果官网无法下载，打开 GitHub Releases 页面，Windows 用户下载 CC-Switch-v3.16.5-Windows-Portable.zip 这个版本。

官网下载 https://www.ccswitch.io/zh/ GitHub Releases 备用下载 无法从官网下载时使用

第二部分

一键导入

一键导入会将 WaterHill 密钥和服务地址直接带入 CC Switch，无需手动填写参数。

打开 WaterHill 的 API 密钥列表，找到要使用的密钥，在操作栏点击“导入到 CCS”。

确认配置无误后，在弹窗右下角点击“导入”完成一键导入。

导入完成后，在供应商列表中找到该供应商，点击“启用”。

第二部分

手动导入

需要自行填写供应商信息时，打开 CC Switch 的 OpenAI 界面，点击右上角“+”添加供应商。

名称可以自行填写；官网链接和 API 请求地址均填写 https://www.waterhill.cyou/v1；API Key 粘贴第一部分创建的 WaterHill 密钥。

保存完成后，在供应商列表中找到该供应商，点击“启用”。

第三部分

下载 Codex

可以直接在 Microsoft Store 搜索下载 Codex，也可以打开下面的 Microsoft Store 链接安装。

Microsoft Store 下载页 安装后回到 CC Switch 选择刚添加的 WaterHill 供应商

第三部分

开始使用

安装并打开 Codex 后，在输入框里输入任务，需要时选择项目，然后点击发送按钮开始使用。

优化使用体验

中文修改

如果希望 Codex 的回答更符合中文使用习惯，可以在新对话开始时直接说明：请使用中文回复，并保持步骤清晰、用词简洁。

1

在任务开头写清语言要求

例如输入“后续请使用中文回答，并把操作步骤写清楚”。这样后面的解释、总结和修改说明都会更适合中文阅读。

2

需要更细的风格时继续补充

可以追加“少用英文术语”“保留命令和文件名原文”“最终回复短一点”等要求，让输出风格更稳定。

优化使用体验

权限批准

Codex 在执行文件修改、联网、安装依赖或运行命令时，可能会根据环境设置请求批准。遇到批准提示时，先确认操作内容再继续。

1

确认命令用途

重点看它要访问的网站、要修改的文件、要运行的命令是否符合当前教程目标。

2

只批准看得懂的操作

如果不确定，可以先让 Codex 解释命令用途，再决定是否批准。涉及密钥、账户、删除文件的操作尤其要谨慎。

优化使用体验

Skill 和插件使用

Skill 和插件可以让 Codex 更擅长处理特定任务，例如文档、表格、演示文稿、图片或项目脚手架。需要时可以直接在任务里点名使用。

1

什么时候使用 Skill

当任务有明确类型时可以直接说，例如“使用文档 Skill 帮我整理 Word 文件”或“使用表格 Skill 分析 Excel”。

2

什么时候使用插件

插件适合扩展 Codex 的工作流。安装或启用后，可以在同一个对话中让 Codex 调用对应能力完成更专门的操作。
